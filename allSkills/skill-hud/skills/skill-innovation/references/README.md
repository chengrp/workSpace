# Skill-Innovation 参考文档

## 创新分析流程

```
1. 读取 skill-match 生成的 skills_data.json
   ↓
2. 加载 ownership_config.json 配置
   ↓
3. 分析每个 skill 的所有权类型
   - 原创：检查 SKILL.md author 字段
   - 魔改：检查 based_on 字段
   - 原生：默认分类
   ↓
4. 计算创新评分
   - 独特性评分
   - 实用性评分
   - 完整性评分
   - 活跃度评分
   ↓
5. 生成魔改推荐
   - 高价值但低活跃度的 skill
   - 功能互补的 skill 组合
   ↓
6. 生成创新指导报告
```

## 所有权判断规则

### 原创 (Original)
满足以下任一条件：
- `ownership_config.json` 中标记为 `"type": "原创"`
- SKILL.md 中 `author` 字段匹配当前用户
- 不在任何公开仓库中找到

### 魔改 (Modified)
满足以下任一条件：
- `ownership_config.json` 中标记为 `"type": "魔改"`
- SKILL.md 中有 `based-on` 或 `forked-from` 字段
- 描述中提到 "基于"、"修改自"

### 原生 (Native)
不满足以上条件，直接使用他人 skill

## 创新评分算法

```python
创新分数 = (
    独特性分数 * 0.3 +
    实用性分数 * 0.3 +
    完整性分数 * 0.2 +
    活跃度分数 * 0.2
)

独特性 = 100 - (相似skill数量 * 10)
实用性 = min(100, 描述长度 / 10 + 功能点数量 * 15)
完整性 = (文档完整度 + 代码规范度) / 2
活跃度 = min(100, 最近更新天数倒数 * 10 + commit数量 * 5)
```

## 魔改推荐策略

### 1. 高价值低活跃度
- 条件：创新分数 > 70 但 6 个月无更新
- 建议：接管维护、添加新功能

### 2. 功能互补组合
- 条件：两个 skill 功能相关但不重叠
- 建议：创建组合 skill 或工作流

### 3. 本地化优化
- 条件：功能通用但可以本地化
- 建议：添加本地配置、优化输出格式
