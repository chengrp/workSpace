# Skill-Health 健康度评估 v2.2

## 功能说明

评估每个 Skill 的健康状态，包括文件结构、元数据完整性、安全性等。

## 评分方案（v2.2 - 平衡区分型）

### 基础分：80 分

只要 SKILL.md 存在且有基本内容（>50行）即可获得基础分。

### 严重问题（大额扣分）

| 问题 | 扣分 | 说明 |
|------|------|------|
| 缺少 SKILL.md | -100 | 直接 0 分 |
| SKILL.md 过短（<50行） | -40 | 内容太少 |
| 安全问题 | 一票否决 | 高风险安全问题 |

### 轻微问题（固定扣分）

| 问题 | 扣分 | 说明 |
|------|------|------|
| 文件过长（>500行） | -5 | 建议拆分 |
| 元数据不完整 | -5 | 缺少必需字段 |
| 元数据警告 | -1 | 缺少可选字段 |
| 缺少 scripts 目录 | -3 | 大部分 skill 不需要 |
| 缺少使用示例 | -3 | 影响理解使用 |
| 有不必要文件 | -3 | .pyc 等 |
| Python 脚本不规范 | -1~3 | 轻微规范问题 |

### 加分项（最高+20分）

| 项目 | 加分 | 说明 |
|------|------|------|
| 有完整 triggers | +5 | 至少 2 个触发词 |
| scripts 规范完善 | +5 | 有代码且规范 |
| 有使用示例 | +4 | 代码块示例 |
| 有 README | +3 | 额外文档 |
| 有 references 目录 | +3 | 参考资料 |
| 版本号规范 | +2 | 语义化版本 |

### 评级标准

| 分数 | 等级 | 说明 |
|------|------|------|
| 90-100 | 🏆 优秀 | 完善的 skill |
| 80-89 | 👍 良好 | 基本完善 |
| 70-79 | 😐 及格 | 满足基本要求 |
| 0-69 | ⚠️ 需改进 | 存在明显问题 |
| 安全问题 | 🚨 不合格 | 有安全风险 |

## 运行方式

### 基础检查
```bash
cd skill-health/scripts
python skill_health.py
```

### 启用安全性深度检查
```bash
cd skill-health/scripts
python skill_health.py --deep-security
# 或
python skill_health.py -d
```

### 查看帮助
```bash
python skill_health.py --help
```

## 输出文件

- `skills_health_data.json` - 所有技能的健康数据
- `../reports/SKILLS_健康度_*.md` - 健康度报告
- `../backups/` - 重构备份（可选）
- `../refactor_logs/` - 重构日志（可选）

## 检查项

### 第一阶段：核心规范

| 检查项 | 说明 | 权重 |
|--------|------|------|
| 触发词完整性 | 检查 triggers 字段是否存在且格式正确 | 10% |
| 版本号规范 | 检查 version 是否符合语义化版本 | 8% |
| MCP 依赖检查 | 检查 MCP 依赖声明 | 包含在元数据中 |

### 第二阶段：质量提升

| 检查项 | 说明 | 权重 |
|--------|------|------|
| 使用示例检查 | 检查是否包含代码块示例 | 10% |
| Python 脚本规范 | 检查 shebang、docstring、`if __name__` | 5% |
| 安全性基础检查 | 硬编码敏感信息（API Key、密码） | **一票否决** |
| 安全性深度检查 | 危险函数（eval、exec、shell注入） | **一票否决** |

### 基础检查

| 检查项 | 说明 | 权重 |
|--------|------|------|
| 文件长度 | SKILL.md 内容长度 | 15% |
| 结构完整性 | 是否有 SKILL.md、scripts/ | 25% |
| 元数据验证 | name、version、description | 17% |
| 代码规范 | scripts/ 目录规范 | 10% |

**总权重：100%**

## 健康等级

- **🏆 优秀** (90-100分): 符合所有规范
- **👍 良好** (80-89分): 基本符合规范
- **😐 及格** (70-79分): 存在部分问题
- **⚠️ 需改进** (0-69分): 需要大幅改进
- **🚨 不合格** (安全问题): 发现高风险安全问题

## 命令行选项

| 选项 | 说明 |
|------|------|
| `-d, --deep-security` | 启用安全性深度检查（可能影响性能） |
| `-h, --help` | 显示帮助信息 |

## 安全性检查

### 基础检查（默认启用）
- 硬编码 API Key
- 硬编码密码
- 硬编码 Token
- 硬编码 Secret Key

### 深度检查（需 --deep-security）
- `eval()` 函数使用
- `exec()` 函数使用
- Shell 命令拼接注入风险

**注意**：深度安全检查可能增加 10-30% 的执行时间，系统会提示用户确认。

## 重构功能

自动重构包括：
1. 自动备份（支持回滚）
2. 清理不必要文件（.pyc、__pycache__、.DS_Store等）
3. 创建推荐文件（README.md等）
4. 创建标准目录结构（scripts/、references/）
5. 生成重构日志（记录所有操作）
