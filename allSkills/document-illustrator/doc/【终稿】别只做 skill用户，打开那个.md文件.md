## 01. Skill 是个白盒，不是魔法

当你打开一个 Skill 的 `.md` 文件，看到的不是代码，是一段思维的快照。

AI应用有白盒和黑盒的区别。ChatGPT 的网页版是黑盒，你不知道它怎么想；而 Skill 是白盒，作者把思考过程摊开给你看。

一个标准的 Skill（按 Anthropic 的架构）就四样东西：

```
SKILL.md       ← 大脑与契约
references/    ← 外脑（文档、规范）
examples/      ← 手脚（模板、样本）
scripts/       ← 脊梁（逻辑、校验）
```

大部分人只盯着 `SKILL.md` 看指令，忽略了后面三个。高手功力藏在那里：

- `references/` 存着文档和规范，平时不加载，用到才读。这叫"渐进式披露"，大脑保持轻快，知识放抽屉里。
- `examples/` 不是模板，是变体。给 AI 看 3 种不同风格，它才不会写出一个模子里刻出来的垃圾。
- `scripts/` 最容易被忽视。Prompt 负责说什么，Script 负责怎么执行。没有这一层，AI 只是会聊天的玩具。

## 02. 偷走高手的认知模型

Skill 拆解不是学语法，是认知逆向工程。

以我最近琢磨的 `wechat_article_skills` 为例。表面上看，它是个"帮我写文章"、公众号排版的工具。你安装应用它，得到的是个不知疲倦的打字员。

但如果你打开文件夹，按标准架构拆解它，实际上是在汲取一位资深运营总监的职业经验。

**拆 SKILL.md，看顶层设计**

第一遍不用一行一行逐一梳理，要快速了解框架是关键。
不入先让AI先生成"内容结构图"与“流程步骤”，要求"产品思维驱动"。

![image.png](http://t8t3msqzv.hn-bkt.clouddn.com/img/20260203091206181.png?imageView2/2/w/1200)


接下来拆流程换对象，装扮为自己喜欢的样子。
完完整整，改造为你的思维链路。

**拆 scripts，掌握硬核验收**

这里藏着 `publisher.py`、`formatter.py`，更重要的是 Validation 脚本。

普通人教 AI "怎么写"，高手写脚本检查"写对了没有"。自动检测：是否有 3 个金句？标题是否 15-20 字？

没有验收的 AI 只是玩具。

**拆 references，复制隐性知识**

`.css` 排版规范、API 文档放在这里，按需读取。
这就是渐进式披露，知识不在脑子里，在手边。

**拆 examples，洞悉样本变体**

这是大部分人的误区。examples 不是模板文件夹，是变体库。

`tech_style.md`（技术风）、`story_style.md`（故事风）、`crisis_pr.md`（危机公关风）。

Variation 是 examples 的灵魂。通过多样样本，你教会了 AI 应对不同场景的能力。

## 03. 天花板破法

所谓"技术平权"，其实不需要懂 Python 语法，不需要啃微信 API 文档，甚至不需要有文案功底。
只需要看懂文件夹结构，就能把行业大牛的最佳实践 Fork 下来。

标准化 Skill 面前，没有技术壁垒。

**第一刀，改大脑（SKILL.md）**

我把原作者的"产品经理视角"改成了自己的风格：

> "你是老王，写东西喜欢用短句、排比，讨厌长篇大论。经常用'我'的视角分享经历，金句偏向总结型，常在段尾。"

瞬间，AI 的输出就像老王本色。
![image.png](http://t8t3msqzv.hn-bkt.clouddn.com/img/20260203091600105.png?imageView2/2/w/1200)

**第二刀，改手脚（scripts）**

加了自己的验收标准。发布前自动检查：是否有违禁词？标题 15-20 字？是否包含至少一个真实案例？

之前发文章经常被平台限流，现在 `scripts`校验脚本自动拦截。

瞬间，拥有不知疲倦的质检员。

**第三刀，改样本（examples）**

替换原作者案例。放入你最高赞的 3 篇文章，一篇干货、一篇情感、一篇种草。

告诉 AI："看，这就是我要的 Variations。"

瞬间，AI 懂得了百变风格，不再只是机器。

## 04. 动手，别光看

到现在，你已经参透了行业大牛的心法。
当别人还在焦虑"怎么写爆款文案"时，你可以：

1. **下载** — 去 GitHub 找开源 Skill
2. **解剖** — 对应 SKILL.md / scripts / references / examples 四大件
3. **魔改** — 在 scripts 植入验收，在 examples 注入变体

"会用" Skill 是及格，"会改" Skill 才是核心竞争力。

到现在，变优秀不用再苦练每一项技能；
你有了更快更成熟的路径——**拆解世界上优秀的技能包，通过结构化文本编辑，重构为我所用**。

别只做 AI 的用户了,
打开那个 .md 文件，拿回你的创造力。
