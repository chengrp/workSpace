## 模板 3：判断句式库

### 核心判断句式（对比型）
```
- 真正的问题不在于[X]，而在于[Y]
- 核心不是"[X]"，而是"[Y]"
- [X]是平等的，但[Y]不是
- 关键不是[X]，而是[Y]
- 重点不在于[X]，而在于[Y]
- [X]很重要，但[Y]更重要
```

### 趋势判断句式（时间型）
```
- 从[A]到[B]，[时间]的[趋势]，和[受众]的第一次[判断]
- [时间]以来，[领域]最大的变化是[X]
- 说到底，[X]不是[过去状态]，而是[未来趋势]
- [时间]过去了，[X]的问题没解决，[Y]的问题又来了
```

### 决定论判断句式（因果型）
```
- [X]的可及性，决定了[Y]
- [X]的水平，决定了[Y]的上限
- 不是[X]决定了[Y]，而是[Y]决定了[X]
- [X]可以救急，但[Y]才能救命
```

### 否定判断句式（反转型）
```
- 说到底，[X]不是[A]，而是[B]
- [X]从来没想过[Y]，它只关心[Z]
- 这不是[X]的问题，这是[Y]的问题
- [X]和[Y]不是一回事，[具体说明]
```

### 程度判断句式（强调型）
```
- [X]好用的前提是[Y]
- [X]难用到离谱
- [X]的效果？有点惨不忍睹
- [X]确实有效，但[Y]是致命的
- [X]贵死了，[Y]倒是便宜
```

### 对比判断句式（并列型）
```
- 一方面[X]，另一方面[Y]
- [A]能做[X]，但[B]能做得更好
- [X]适合[A]，[Y]适合[B]，别混了
- [X]解决不了[A]的问题，[Y]才能
```

### 经验判断句式（实战型）
```
- 我试过[X]，确实有效
- 我自己也踩过这个坑
- 说实话，[X]这点让我很不爽
- 我觉得这个产品挺好用，但[Y]让我很失望
```

### 反问判断句式（引导型）
```
- 为什么[X]往往不是[Y]？
- [X]真的能[Y]吗？
- 这是什么意思？[解释]
- 怎么办？[给出方案]
```

## 模板 4：三遍审校检查清单

系统化地降低 AI 检测率至 20% 以下的完整操作清单。

---

### 第一遍：内容审校

#### 事实准确性
- [ ] 所有数据都有来源（真实的，不是编造的）
- [ ] 时间正确（年份、月份、顺序）
- [ ] 产品名称正确（大小写、版本号）
- [ ] 人名和公司名正确
- [ ] 技术术语使用准确
- [ ] 链接有效可访问

**常见错误**:
- ❌ 编造数据：「使用量提升了 85.7%」（没有来源）
- ❌ 时间错误：说 2024 年的事件发生在 2023 年
- ❌ 名称错误：「Claude AI」→「Claude」
- ❌ 术语错误：混淆概念

**修改标准**:
- 所有数据必须有真实来源
- 不确定的事实必须删除或标注
- 宁可少写，不要编造

---

#### 逻辑清晰性
- [ ] 前后无矛盾（前面说 A，后面不能说非 A）
- [ ] 因果关系合理（因为 X 所以 Y 说得通）
- [ ] 论据支持论点（例子能证明观点）
- [ ] 结构完整（有引入、展开、总结）

**常见问题**:
- ❌ 自相矛盾：前面说「很简单」，后面说「需要深厚功底」
- ❌ 因果不当：「因为用了 AI，所以成功了」（太简化）
- ❌ 论据不足：只有一个例子就下普遍性结论

**修改标准**:
- 逻辑链条完整
- 论据充分
- 避免绝对化表述

---

#### 结构合理性
- [ ] 小标题概括段落内容
- [ ] 各部分长度均衡（不要某部分特别长）
- [ ] 没有跑题（所有内容围绕主题）
- [ ] 重要信息前置（不要把核心内容放在最后）

**常见问题**:
- ❌ 小标题和内容不符
- ❌ 某部分占了文章一半篇幅
- ❌ 开头铺垫太长，核心内容太少

**修改标准**:
- 结构清晰，一目了然
- 重点突出
- 详略得当

**AI味估计**：~50%

---

### 第二遍：风格审校

这是降 AI 味的关键一遍！

#### 1. 删除 AI 套话

**必删清单**:
```
❌ 在当今时代
❌ 随着科技的发展
❌ 在...的背景下
❌ 值得注意的是
❌ 需要指出的是
❌ 综上所述
❌ 总而言之
❌ 让我们来看看
❌ 首先让我们
❌ 接下来让我们
❌ 在实际应用中
❌ 从某种意义上说
❌ 毋庸置疑
❌ 不言而喻
❌ 这里有个细节很关键
❌ 这是个本质区别
❌ 更好的策略是
```

**替换对照**:
| 删除 | 替换为 |
|------|--------|
| 值得注意的是 | 注意 / 重点来了 / [直接说内容] |
| 综上所述 | 总结一下 / 说白了 / 简单来说 |
| 让我们来看看 | 我们来看 / 看看 / 说说 |
| 在实际应用中 | 实际用下来 / 我用的时候 |
| 这是个本质区别 | 这两个完全不是一回事 |
| 更好的策略是 | 换个活法 / 这么干 |
| 解决方法 | 怎么办？ |

---

#### 2. 拆解 AI 句式

**过度对比**:
```
❌ 不是 A 而是 B，不是 C 而是 D，不是 E 而是 F
✅ 不是 A，而是 B（只用一次即可）
```

**过度并列**:
```
❌ 既...又...不仅...而且...同时...并且...
✅ 拆成多个短句，每句一个要点
```

**过度排比**:
```
❌ 从 A 到 B，从 B 到 C，从 C 到 D，从 D 到 E
✅ 直接列举：A、B、C、D、E
```

**工整排比**:
```
❌ 要么...要么...要么...
✅ 独立短句 + 问号
例如：「商业化产品？贵死了。学术模型？普通人根本用不了。」
```

**前者后者**:
```
❌ 前者是 XX，后者是 YY
✅ 「第一句话是 XX。第二句话，我一听就知道 YY。」
```

---

#### 3. 替换书面词汇

**常见书面词汇对照表**:

| 书面词汇 | 口语化替换 |
|---------|-----------|
| 显著提升 | 提升了 XX% / 涨了很多 / 效果很好 |
| 大幅增长 | 涨了 X 倍 / 增长了 XX% |
| 充分利用 | 用好 / 用上 / 多用 |
| 有效降低 | 降低 / 减少 |
| 深入了解 | 搞清楚 / 弄明白 / 了解 |
| 进行操作 | 操作 / 做 / 执行 |
| 实施方案 | 执行方案 / 按方案做 / 落地 |
| 开展工作 | 开始做 / 动手 / 推进 |
| 积极响应 | 响应 / 行动 / 参与 |
| 持续优化 | 一直在优化 / 不断改进 |
| 切实可行 | 可行 / 靠谱 / 能做 |
| 行之有效 | 有效 / 管用 / 好用 |
| 很难突围 | 进去就是送 |
| 关键是找到 | 往死里挖 |
| 不要贪大 | 别贪大 |
| 信息密度极高 | 有多专业？ |
| 这点我深有体会 | 我自己也踩过这坑 |

---

#### 4. 改成口语化

**被动语态 → 主动语态**:
```
❌ 这个问题被我们解决了
✅ 我们解决了这个问题
```

**名词化 → 动词化**:
```
❌ 进行了优化
✅ 优化了
```

**长定语 → 拆成短句**:
```
❌ 基于多年产品经验并结合最新 AI 技术研发的这款产品
✅ 这款产品基于多年产品经验。我们结合了最新 AI 技术。
```

---

#### 5. 加入真实细节

**抽象 → 具体**:
```
❌ 效果很显著
✅ 用户量从 1000 涨到 10000，涨了 10 倍

❌ 用户反馈很好
✅ 100 个用户中，95 个说好用

❌ 开发很快
✅ 两个小时就做完了

❌ 功能很强大
✅ 原来要 1 小时的任务，现在 20 分钟搞定
```

---

#### 6. 加入个人态度

**中立 → 有态度**:
```
❌ 这个产品有优点也有缺点
✅ 我觉得这个产品挺好用，但 XX 这点让我很不爽

❌ 这个方法可能有效
✅ 我试过，确实有效

❌ 可以考虑使用
✅ 我推荐用这个

❌ 成本较高
✅ 贵死了

❌ 门槛较高
✅ 门槛高到离谱

❌ 难以使用
✅ 普通人根本用不了

❌ 维护成本增加
✅ 维护起来要命

❌ 可能存在问题
✅ 八成是错的

❌ 效果不佳
✅ 有点惨不忍睹
```

**态度表达词**:
- 「我觉得...」
- 「在我看来...」
- 「说实话...」
- 「老实说...」
- 「我用下来...」
- 「让我很不爽」
- 「挺好用」
- 「很喜欢」

---

#### 7. 删除过度修饰

**过度形容词**:
```
❌ 非常显著的大幅度提升
✅ 提升很大 / 提升了 10 倍

❌ 极其重要的关键因素
✅ 关键因素 / 重要的是
```

**过度副词**:
```
❌ 充分地、有效地、显著地
✅ 删除副词，保留动词
```

**AI味估计**：~25%

### 第三遍：细节打磨

#### 1. 句子长度

**检查标准**:
- [ ] 主流句子 15-25 字
- [ ] 最长不超过 30 字
- [ ] 超过 30 字的长句全部拆短

**拆句方法**:
```
❌ 这个功能不仅提升了效率，而且降低了成本，同时还改善了用户体验，让用户满意度大幅提升。(36字)

✅ 这个功能提升了效率，降低了成本。用户体验也改善了。满意度涨了很多。(3句，分别14/9/8字)
```

---

#### 2. 段落长度

**检查标准**:
- [ ] 手机屏幕 3-5 行
- [ ] 约 100-150 字
- [ ] 长段落拆分

**拆段方法**:
- 一个段落只说一件事
- 换话题就分段
- 重要观点单独成段

---

#### 3. 标点使用

**句号**:
- 多用句号
- 不要用逗号连接多个独立句子
- 一句话一个完整意思就用句号

**逗号**:
- 只在句内停顿时用
- 不要连接 3 个以上的短句
- 连接 2 个短句就够了

**问号**:
- 适当提问，引起思考
- 不要连续多个问句
- 问完要给答案

**感叹号**:
- 只在真正需要强调时用
- 不要滥用
- 一段最多 1-2 个

**引号规范**:
- 使用中文直角引号「」
- 双重引用使用『』
- 不使用英文引号 "" 或 ''

---

#### 4. 节奏变化

**长短句交替**:
```
短句。短句。长一点的句子，有些补充说明。又是短句。
```

**快慢结合**:
- 重要信息：快节奏，短句
- 解释说明：慢节奏，适当加长
- 高潮部分：短句连发

---

#### 5. 排版美观

**留白**:
- [ ] 段落之间空一行
- [ ] 重要观点前后留白
- [ ] 不要密密麻麻

**小标题**:
- [ ] 层级清晰（## → ###）
- [ ] 概括内容
- [ ] 长度适中（不超过 15 字）

**列表**:
- [ ] 对齐
- [ ] 格式统一（- 或 1.）
- [ ] 每项长度相近

**AI味估计**：~20%

### AI 味检测终极清单

完成三遍审校后，最后检查：

#### 语言层面
- [ ] 删除了所有 AI 套话（「在当今」「综上」等）
- [ ] 没有过度对比和排比
- [ ] 替换了所有书面词汇（「显著」「充分」等）
- [ ] 改成了口语化表达
- [ ] 句子长度合适（15-25字）

#### 内容层面
- [ ] 有真实数据（不是编造的）
- [ ] 有具体细节（不是抽象描述）
- [ ] 有个人经历和观点
- [ ] 态度明确（不是中立客观）

## 模板 5：标题选择标准

### 5种标题类型

| 类型 | 特点 | 示例 |
|------|------|------|
| **疑问型** | 提出问题，引发好奇 | 《XXX 真的能 YYY 吗？》 |
| **对比型** | 对比两个选项 | 《A vs B：哪个更好？》 |
| **数据型** | 用数据吸引注意 | 《XXX 提升 200% 的秘密》 |
| **指南型** | 承诺提供方法 | 《如何用 XXX 做 YYY》 |
| **故事型** | 个人经历引入 | 《我用 XXX 的 30 天》 |

### 标题检查清单
- [ ] 标题能吸引目标读者？
- [ ] 标题准确反映文章内容？
- [ ] 标题长度合适（15-25字）？
- [ ] 标题没有过度夸张？
- [ ] 标题包含关键词？
