## 完整工作流程示例

### 第一步：内容分析

当用户请求生成信息图时，Claude 会：

1. **分析内容类型**：技术评测、教程、数据报告、产品对比等
2. **识别关键信息**：主题、关键词、目标平台、特殊要求
3. **检测语言**：自动检测中文/英文/混合，确保语言保持

### 第二步：风格推荐

根据内容分析，推荐最匹配的风格：

| 内容类型 | 推荐风格 | 理由 |
|---------|---------|------|
| 技术评测/产品对比 | 1-坐标蓝图 | 技术图纸感，数据对比清晰 |
| 小红书爆款/复古风格 | 2-复古波普网格 | 70年代复古，粗描边平涂 |
| 文具美学/剪贴板 | 3-文件夹风格 | 3D剪贴板，索引标签 |
| 收据/票务美学 | 4-打印热敏纸风 | 现代拟物设计，穿孔边缘 |
| 手账/3D图标 | 5-复古手帐风 | 手绘细节，双语分层 |
| 侦探/档案/拼贴 | 6-档案风 | 证据板，混合媒体 |
| 现代几何/色块 | 7-色块风 | 高对比度，扁平设计 |
| 真实票据/印刷 | 8-票据风 | 热敏印刷，票据编号 |

### 第三步：模块规划

将内容提炼为 6-7 个核心模块：

- **技术对比**：参数、价格、能力、场景、优缺点
- **教程指南**：步骤、要点、案例、技巧
- **数据报告**：规模、趋势、分析、结论
- **故事叙述**：人物、情节、线索、反转

### 第四步：Prompt 生成

自动生成包含以下元素的完整 Prompt：

- **色彩规范**：背景色、主色、强调色
- **布局要求**：高密度 6-7 模块，无空白
- **视觉元素**：图标、装饰、风格特征
- **语言保持**：中文/英文/混合保持原样
- **输出规格**：尺寸比例、质量要求

### 第五步：生成图片

使用生成的 Prompt 调用文生图 API：

**Google Generative AI 示例：**
```python
import google.generativeai as genai
import os

api_key = os.environ.get("ALL_IMAGE_GOOGLE_API_KEY")
genai.configure(api_key=api_key)

model = genai.GenerativeModel("gemini-2.0-flash-exp-image-generation")
response = model.generate_content("生成的 Prompt")

with open("output.png", "wb") as f:
    f.write(response.data)
```

## 提示

1. **语言保持**：本技能自动检测并保持原始语言，中文内容不会被翻译
2. **模块数量**：确保提炼 6-7 个模块，少于 6 个会显得稀疏
3. **风格匹配**：根据内容类型选择最合适的风格
4. **质量选择**：小红书推荐 4K 质量，社交媒体可用 High
5. **比例选择**：竖版 3:4 适合小红书，横版 16:9 适合公众号
