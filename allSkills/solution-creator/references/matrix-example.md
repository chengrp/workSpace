## 产品矩阵使用示例

### 示例 1：客户核心痛点是"文档检索慢"

**匹配过程**：
1. 在"对应痛点"列中搜索关键词"检索"
2. 找到"智能知识库"功能模块
3. 提取"核心价值主张"：检索效率提升 70%，从 2 小时降至 5 分钟
4. 提取"竞品差异化"：私有化部署、不限 Token
5. 提取"实施依赖"：需要 GPU 服务器、需要提供运维文档

**方案输出**：
> "针对您提到的文档检索慢的问题（平均耗时 2 小时），我们通过智能知识库，帮您实现知识的即搜即用，预计检索效率提升 70%（从 2 小时降至 5 分钟）。与竞品相比，我们支持私有化部署，数据完全自主可控，且不限 Token 数量，无额外成本。实施前需要准备 GPU 服务器和近 3 年的运维文档。"

### 示例 2：客户关心"成本问题"

**匹配过程**：
1. 在"核心价值主张"列中搜索关键词"成本"、"节省"
2. 找到多个功能模块：
   - 自动巡检 Agent：每年节省 80 万元
   - 容量预测引擎：每年节省 50 万元
3. 累计成本节约：130 万元/年

**方案输出**：
> "通过自动巡检 Agent 和容量预测引擎的组合应用，预计每年可为您节省人力和硬件成本约 130 万元，投资回报周期约 6 个月。"

## 填写要点总结

1. **痛点要具体**：使用量化数据（时间、成本、错误率）
2. **价值要清晰**：从"现状"到"目标状态"，带上提升幅度
3. **差异化要明确**：避免"业界领先"等模糊表述，用具体特点说话
4. **依赖要完整**：技术、数据、环境、人力、第三方都要考虑到
