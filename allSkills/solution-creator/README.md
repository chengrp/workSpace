# Solution Creator - 解决方案生成器

基于需求解码结果，结合产品矩阵，生成高价值、高维度的解决方案初稿。

## 功能概述

Solution Creator 是一个专业的 B2B 解决方案生成工具，通过三段式工作流将产品能力转化为业务价值：

1. **价值锚定**：建立需求痛点与产品能力的精准映射
2. **拔高升华**：将具体需求上升到战略高度
3. **实施审计**：确保方案落地可行性，识别风险点

## 快速开始

### 1. 填写产品矩阵

复制 `assets/product-matrix.md` 中的模板，根据你的产品情况填写：

```markdown
| 功能模块 | 对应痛点 | 核心价值主张 | 竞品差异化 | 实施依赖 |
| :--- | :--- | :--- | :--- | :--- |
| **[功能名称]** | [解决什么问题] | [带来什么价值] | [相比竞品的独特优势] | [需要什么前提条件] |
```

**填写要点**：
- 痛点要具体：使用量化数据（时间、成本、错误率）
- 价值要清晰：从"现状"到"目标状态"，带上提升幅度
- 差异化要明确：避免"业界领先"等模糊表述
- 依赖要完整：技术、数据、环境、人力都要考虑到

### 2. 使用技能

当用户请求以下任一任务时，技能自动触发：

- "基于这个需求生成解决方案"
- "为这个项目写一份方案建议书"
- "把需求解码表转化为解决方案"
- "匹配产品能力生成方案"
- "编写投标方案"
- "制定数字化转型规划"

### 3. 输出内容

技能按照以下顺序输出：

1. **[方案生成逻辑链]**：说明匹配了哪些痛点，运用了什么拔高策略
2. **[待确认的实施红线]**：列出风险点和待确认项
3. **[解决方案建议书]**：完整的方案文档（遵循 `proposal-template.md` 结构）

## 核心资产说明

### product-matrix.md

产品矩阵是方案生成的"弹药库"。所有方案内容都基于此矩阵，AI 不会凭空编造。

**关键列**：
- **对应痛点**：描述客户面临的具体问题（带量化数据）
- **核心价值主张**：从功能转化为价值（带提升幅度）
- **竞品差异化**：相比竞品的独特优势（技术/性能/成本/合规）
- **实施依赖**：需要的前提条件（硬件/软件/数据/人力）

### proposal-template.md

方案建议书的"交付模具"，包含以下章节：

1. **建设背景与愿景**：将需求上升到战略高度
2. **现状与痛点分析**：清晰呈现当前问题
3. **总体解决方案**：功能-价值映射
4. **实施保障与前提**：列出实施红线
5. **预期收益 ROI**：量化收益说明
6. **附录：产品技术规格**：技术细节

## 使用场景

### 场景 1：从需求解码表生成方案

**用户输入**：
> "基于这份需求解码表，生成一份解决方案。客户核心痛点是文档检索慢和人工巡检成本高。"

**技能输出**：
1. 读取产品矩阵，匹配对应功能
2. 拔高：构建"AI 驱动的智能运维体系"
3. 输出：完整方案建议书 + 实施红线清单

### 场景 2：竞品对比场景

**用户输入**：
> "客户提到竞品也有类似功能，如何突出我们的优势？"

**技能输出**：
在"功能-价值映射图"中强化"竞品差异化"列，例如：
- 竞品："公有云部署，按 Token 计费"
- 我方："私有化部署，不限 Token，数据完全自主可控"

### 场景 3：成本敏感客户

**用户输入**：
> "客户很关心成本，如何突出性价比？"

**技能输出**：
在"预期收益 ROI"章节中：
- 列出所有成本节约项（人力、硬件、第三方服务）
- 计算投资回报周期（如：6 个月收回成本）
- 对比竞品的总拥有成本（TCO）

## 核心原则

### 价值导向
- ❌ "卖功能"：我们有智能搜索功能
- ✅ "卖价值"：实现知识的即搜即用，效率提升 70%

### 战略拔高
- ❌ "修修补补"：帮您做个自动巡检
- ✅ "战略升级"：构建主动预防的智能运维体系

### 风险透明
- ❌ "盲目承诺"：什么都能做，没有限制
- ✅ "风险透明"：清晰标注所有实施依赖和待确认项

### 证据确凿
- ❌ "模糊表述"：性能业界领先
- ✅ "量化指标"：响应时间 < 100ms，比竞品快 5 倍

## 常见问题

### Q1：产品矩阵应该多久更新一次？

**A**：建议每季度至少更新一次，或者在以下情况更新：
- 新功能上线
- 性能指标有显著提升
- 竞品情况发生变化

### Q2：如何平衡"拔高"和"落地"？

**A**：
- **拔高**：在"建设背景与愿景"章节体现战略高度
- **落地**：在"实施保障与前提"章节列清所有依赖
- 两者的平衡是方案专业性的体现

### Q3：如果产品矩阵中没有对应功能怎么办？

**A**：
- **方案生成阶段**：明确标注"需定制开发"，并评估人天
- **产品规划阶段**：将缺失功能反馈给产品团队
- **绝对禁止**：凭空编造不存在的功能

### Q4：如何确保方案的可信度？

**A**：
1. 所有数据都来自产品矩阵，不凭空捏造
2. 价值主张都有量化指标支撑
3. 实施依赖清晰标注，不隐瞒风险
4. 竞品差异化有具体依据，不使用模糊表述
