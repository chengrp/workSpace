# 核心产品价值矩阵 (Product Value Matrix)

> **使用说明**：这是方案生成器的"弹药库"。请根据你的实际产品情况填写此表格。AI 生成的所有方案都将基于此矩阵中的信息。

## 产品矩阵模板

| 功能模块 (Feature) | 对应痛点 (Pain Point) | 核心价值主张 (Value Proposition) | 竞品差异化 (Differentiator) | 实施依赖 (Implementation Dependency) |
| :--- | :--- | :--- | :--- | :--- |
| **[功能名称]** | [解决什么问题] | [带来什么价值] | [相比竞品的独特优势] | [需要什么前提条件] |

## 填写指南

### 1. 功能模块 (Feature)
- 使用清晰的名称，如"智能知识库"、"自动巡检 Agent"
- 可按产品模块分层级

### 2. 对应痛点 (Pain Point)
- 描述客户当前面临的具体问题
- 量化问题的影响（如"耗时 2 小时"、"成本增加 30%"）

### 3. 核心价值主张 (Value Proposition)
- **从功能到价值的转化**：
  - ❌ "提供智能搜索功能"
  - ✅ "实现知识的即搜即用，效率提升 70%"
- 尽可能量化价值

### 4. 竞品差异化 (Differentiator)
- **差异化维度**：
  - 技术架构（如：私有化部署 vs 公有云）
  - 性能指标（如：响应速度、并发量）
  - 成本优势（如：不限 Token、按需付费）
  - 合规性（如：数据自主可控）
  - 独特能力（如：独有的算法、专利技术）

### 5. 实施依赖 (Implementation Dependency)
- **技术依赖**：需要 GPU 服务器、需对接特定 API
- **数据准备**：需要客户提供历史数据、需要数据清洗
- **环境要求**：需要内网环境、需要特定权限
- **人力投入**：需要培训、需要专人维护
