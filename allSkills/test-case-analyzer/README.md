# 测试用例分析器 (Test Case Analyzer)

> 从需求文档自动生成专业测试用例，支持学习进化

[![Version](https://img.shields.io/badge/version-1.0.0-blue.svg)](SKILL.md)
[![Status](https://img.shields.io/badge/status-MVP-green.svg)](CHANGELOG.md)

---

## 简介

测试用例分析器是一个 Claude Code Skill，能够基于专业的需求分析方法论，从需求文档自动生成结构化的测试用例。

### 核心特性

- **五维度分析** - 基于专业测试工程方法论
- **多格式输入** - 支持 Markdown, PDF, DOCX, PPTX
- **双格式输出** - Markdown + Excel 同时生成
- **学习进化** - 根据反馈持续改进

## 使用示例

### 示例 1：电商订单需求

**输入**：
```markdown
# 订单超时自动关闭功能

为提升库存周转率，用户下单后超过 30 分钟未支付，
系统将自动关闭订单并释放库存。
```

**输出**：
- 功能测试：30分钟后自动关闭
- 边界值测试：29分59秒支付
- 并发测试：支付与关闭同时发生
- 状态测试：订单状态流转
- 回归测试：库存数据同步

### 示例 2：用户权限需求

**输入**：
```markdown
# 角色权限管理

系统支持三种角色：管理员、编辑者、查看者。
管理员可以管理用户，编辑者可以创建内容，
查看者只能浏览内容。
```

**输出**：
- 功能测试：各角色权限验证
- 边界值测试：权限边界
- 状态测试：角色切换
- 安全测试：越权访问

## 输出格式

### Markdown 示例

```markdown

- 待确认事项：计时单位、实现方式
```

生成 4 个 Sheet：
1. **测试用例总表** - 所有测试用例
2. **需求分析摘要** - 分析结果
3. **待确认事项** - 需要澄清的问题
4. **测试范围评估** - 测试覆盖分析

## 需求分析方法论

基于**五维度分析框架**：

1. **核心要素和业务流程**
   - 触发条件、计时条件、业务流程

2. **判断和执行条件**
   - 执行动作、判断时机、处理方式

3. **预期结果**
   - 状态变化、数据变更、用户感知

4. **异常流程分析**
   - 边界值、并发问题、业务交互

5. **外部依赖和配置**
   - 可配置参数、外部服务、特殊场景

## 学习与进化

每次使用后，系统会询问反馈：

1. **覆盖率** - 是否遗漏测试场景？
2. **准确性** - 用例描述是否准确？
3. **可用性** - 步骤是否清晰？

反馈数据存储在 `evolution.json`，用于持续改进。
