# 用户角色权限管理

## 背景

系统需要支持不同角色的用户访问控制，确保数据安全和功能隔离。

## 需求描述

系统支持三种用户角色，每种角色拥有不同的权限：

### 角色定义

| 角色 | 描述 |
|------|------|
| 管理员 | 拥有所有权限，可以管理用户和系统配置 |
| 编辑者 | 可以创建和编辑内容，但不能管理用户 |
| 查看者 | 只能浏览内容，不能进行任何修改操作 |

### 权限矩阵

| 功能 | 管理员 | 编辑者 | 查看者 |
|------|--------|--------|--------|
| 浏览内容 | ✅ | ✅ | ✅ |
| 创建内容 | ✅ | ✅ | ❌ |
| 编辑内容 | ✅ | ✅ | ❌ |
| 删除内容 | ✅ | ❌ | ❌ |
| 管理用户 | ✅ | ❌ | ❌ |
| 系统配置 | ✅ | ❌ | ❌ |

## 核心规则

1. 用户只能被分配一个角色
2. 角色变更后，权限立即生效
3. 跨越权限的操作将被拒绝
4. 敏感操作需要二次验证
