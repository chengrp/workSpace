# 积分兑换系统需求文档

## 版本信息
- **版本号**: v1.0
- **创建日期**: 2025-12-15
- **产品**: 会员积分系统

## 背景

为提升用户活跃度和平台粘性，需要开发一套完整的积分兑换系统。用户通过日常活动获得积分，可以使用积分兑换平台上的虚拟商品、优惠券等。

## 需求概述

### 核心功能

1. **积分获取**: 用户完成指定任务后自动获得积分
2. **积分消耗**: 用户使用积分兑换商品时扣除相应积分
3. **积分过期**: 用户积分有效期为获得后 365 天，过期自动清零
4. **积分记录**: 完整记录用户积分的获取、消耗、过期等变动历史

## 功能需求

### 1. 积分获取规则

| 任务类型 | 积分值 | 每日上限 | 说明 |
|---------|--------|---------|------|
| 每日登录 | +5 | 5 | 当天首次登录获得 |
| 完成订单 | +订单金额×1 | 无上限 | 订单完成（非退款状态）后获得 |
| 邀请好友 | +100 | 100 | 好友完成首单后获得 |
| 签到 | +1 | 1 | 连续签到有额外奖励 |

**特殊规则**:
- 连续签到 7 天，第 7 天额外奖励 20 积分
- 新用户注册即送 100 积分

### 2. 积分兑换功能

#### 2.1 商品管理

管理员可以在后台配置可兑换商品：

- **虚拟商品**: 优惠券、会员权益、虚拟道具
- **实物商品**: 需要填写收货地址，通过物流发货

每个商品包含：
- 商品名称
- 所需积分
- 库存数量
- 兑换条件（如：仅限特定等级用户）
- 有效期

#### 2.2 兑换流程

1. 用户浏览积分商城，选择商品
2. 确认兑换信息（所需积分、商品详情）
3. 点击兑换按钮
4. 系统校验：
   - 用户积分是否足够
   - 商品库存是否充足
   - 用户是否满足兑换条件
5. 校验通过后：
   - 扣除用户积分
   - 减少商品库存
   - 生成兑换订单
   - 发送兑换成功通知

### 3. 积分过期机制

- **积分有效期**: 从获得之日起 365 天
- **过期处理**: 每天凌晨批量清理过期积分
- **过期提醒**: 积分过期前 7 天、3 天、1 天推送提醒通知

### 4. 积分查询

用户可以查看：
- 当前积分余额
- 积分明细记录（获取/消耗/过期）
- 即将过期的积分

## 非功能需求

### 性能要求

- 积分变动响应时间 < 500ms
- 支持 10,000 QPS 的并发积分查询
- 积分过期批处理在 30 分钟内完成

### 安全要求

- 防刷积分机制（异常行为检测）
- 积分变动操作记录审计日志
- 兑换操作需要二次确认（高价值商品）

## 数据模型

### 用户积分表 (user_points)

| 字段 | 类型 | 说明 |
|------|------|------|
| user_id | bigint | 用户ID |
| total_points | int | 当前总积分 |
| expired_points | int | 已过期积分 |

### 积分明细表 (points_detail)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键ID |
| user_id | bigint | 用户ID |
| change_type | tinyint | 变动类型：1=获取 2=消耗 3=过期 |
| points | int | 变动积分（正为增加，负为减少） |
| reason | varchar(200) | 变动原因 |
| expire_time | datetime | 过期时间 |
| create_time | datetime | 创建时间 |
