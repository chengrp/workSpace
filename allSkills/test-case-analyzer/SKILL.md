## 功能概述

从需求文档自动生成专业测试用例，支持：

| 功能 | 说明 |
|------|------|
| **多格式输入** | Markdown, TXT, PDF, DOCX, PPTX, XLSX |
| **智能分析** | 基于五维度需求分析方法论 |
| **模式匹配** | 自动识别需求类型，应用行业测试模式 |
| **分层输出** | 功能/边界/状态/回归/交互/配置 |
| **双格式输出** | Markdown + Excel (4个工作表) |
| **学习进化** | 根据反馈持续改进 |
| **批量处理** | 支持批量处理整个目录 |
| **命令行** | analyze/evolve/convert/batch 多命令 |

## 需求分析方法论

基于视频转录总结的五维度分析框架：

### 1. 核心要素和业务流程
- **触发条件**: 什么事件启动流程？
- **计时条件**: 时间限制、计时起点、计时单位
- **业务流程**: 正常流程的完整路径

### 2. 判断和执行条件
- **执行动作**: 满足条件后做什么？
- **判断时机**: 如何判断条件已满足？
- **处理方式**: 技术实现方案

### 3. 预期结果
- 状态变化、数据变更、用户感知

### 4. 异常流程分析
- **边界值问题**: 极值场景
- **并发问题**: 同时操作
- **业务交互**: 状态冲突

### 5. 外部依赖和配置
- 可配置参数、外部服务、特殊场景

## 使用流程

### 命令行使用

```bash
cd test-case-analyzer/scripts

# 分析单个文档
python main.py analyze ../examples/电商订单需求.md

# 分析并指定输出目录
python main.py analyze ../examples/订单需求.pdf -o ./output

# 批量处理目录
python main.py batch ../requirements -o ./output

# 查看学习摘要
python main.py evolve --summary

# 查看已知模式
python main.py evolve --patterns

# 导出学习数据
python main.py evolve --export ./my_evolution.json

# 转换文档格式
python main.py convert document.docx -o document.md
```

### 基本使用（自然语言）

```bash
# 1. 提供需求文档
用户: "分析这个需求：path/to/requirement.md"

# 2. 自动检测格式并转换（如需要）
Claude: [检测到 PDF，正在转换...]

# 3. 执行分析
Claude: [识别需求类型 -> 五维度分析 -> 应用学习模式 -> 生成用例]

# 4. 输出结果
Claude:
  - Markdown: output/markdown/需求名称_测试用例.md
  - Excel: output/excel/需求名称_测试用例.xlsx

# 5. 收集反馈
Claude: "请检查：
  1. 是否有遗漏的测试场景？
  2. 用例描述是否准确？
  3. 步骤是否清晰可执行？"

# 6. 记录学习
[用户反馈 -> 更新 evolution.json]
```

### 进化触发

```bash
# 触发学习总结
用户: "/evolve" 或 "复盘一下刚才的分析"

# 输出学习到的模式
Claude:
  - 需求类型: 订单超时
  - 匹配关键词: 订单, 超时, 关闭, 支付
  - 学习场景: 边界值测试 (29分59秒), 并发支付与关闭
  - 应用结果: 已添加 2 个回归测试用例
```

## 学习与进化机制

### 反馈收集

每次生成后询问用户三个维度：

1. **覆盖率** - 是否遗漏重要测试场景？
2. **准确性** - 测试用例描述是否准确？
3. **可用性** - 测试步骤是否清晰可执行？

### 经验持久化

反馈数据存储在 `evolution.json`：

```json
{
  "version": "1.0",
  "patterns": {
    "ecommerce": {
      "订单超时": {
        "missed_scenarios": ["边界值：29分59秒支付"],
        "auto_add": true
      }
    }
  },
  "user_corrections": [...]
}
```

### 智能改进

- **模式识别**: 学习特定类型需求的测试模式
- **缺失提醒**: 记录用户补充的点，下次自动提示
- **行业惯例**: 按行业加载不同测试重点

## 最佳实践

### 需求分析技巧

1. **关注触发条件**: 找出什么事件启动流程
2. **识别时间约束**: 注意计时相关的需求
3. **分析状态机**: 理解系统状态如何流转
4. **考虑并发场景**: 同时操作可能的问题
5. **检查可配置项**: 需要验证配置的范围

### 测试用例编写

1. **用例ID规范**: TC-{类型}-{序号}
   - F: 功能测试
   - B: 边界值测试
   - S: 状态测试
   - R: 回归测试
   - I: 交互测试
   - C: 配置测试

2. **优先级定义**:
   - P0: 核心功能，必须通过
   - P1: 重要功能，影响体验
   - P2: 一般功能，可以延后
   - P3: 边缘场景，时间允许

3. **步骤描述**:
   - 使用动作动词
   - 步骤之间逻辑清晰
   - 包含预期结果

## 待确认事项模板

分析需求时，始终检查以下问题：

- [ ] 计时单位（分钟/秒）
- [ ] 计时实现方式（定时任务/消息队列）
- [ ] 是否可配置
- [ ] 并发处理机制
- [ ] 状态流转规则
- [ ] 外部依赖服务
- [ ] 特殊场景（重启、断网）
